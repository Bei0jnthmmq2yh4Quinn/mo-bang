# 行业热点检索规则（v1）

目标：用户选什么行业，系统就返回什么行业相关热点；不再把全站泛热点直接塞给用户。

## 1. 核心原则
- 行业选择不是展示字段，是**强检索条件**。
- 默认返回**行业热点**，不是全站热点。
- 全站热点只能作为候选池，**必须经过行业匹配过滤**后才能进入结果。
- 结果宁可少，也不要乱。

## 2. 检索流程
### 第一步：行业词扩展
把用户输入行业，扩展成「行业主词 + 子类词 + 经营词 + 用户词 + 风险词」。

以“餐饮”为例：
- 行业主词：餐饮、餐饮行业、餐饮老板、餐饮店、门店
- 子类词：火锅、烧烤、奶茶、茶饮、咖啡、烘焙、快餐、小吃、酒馆、中餐、西餐
- 经营词：客流、到店、团购、外卖、翻台、复购、门头、定价、套餐、活动、引流
- 用户词：顾客、消费者、排队、下单、探店、差评、回头客
- 风险词：倒闭、涨价、闭店、亏损、成本、房租、人工、食材

### 第二步：拼搜索 query
不是搜“热点”，而是按行业拼具体 query：
- 餐饮 行业热点
- 餐饮 行业趋势
- 餐饮 老板
- 餐饮 门店经营
- 餐饮 客流
- 餐饮 外卖
- 餐饮 团购
- 火锅 热点
- 奶茶 热点
- 咖啡 热点
- 餐饮 涨价
- 餐饮 闭店
- 餐饮 消费趋势

### 第三步：结果过滤
标题 / 摘要 / 标签 / 关键词，只要**没有命中行业词库**，直接剔除。

以餐饮为例，至少命中以下任一词才保留：
- 餐饮 / 门店 / 到店 / 客流 / 外卖 / 团购 / 翻台 / 复购
- 火锅 / 烧烤 / 奶茶 / 茶饮 / 咖啡 / 烘焙 / 快餐 / 小吃
- 菜品 / 定价 / 套餐 / 探店 / 顾客 / 消费者

### 第四步：打分排序
按下面维度打分：
- 行业词命中数（越多越高）
- 是否命中强相关经营词（客流、到店、外卖、团购、复购等）
- 是否命中子行业词（火锅、奶茶、咖啡等）
- 热度值 / 排名
- 是否适合内容转化（能不能写成老板会关心的话题）

### 第五步：兜底
如果行业过滤后结果为空：
- 不返回泛热点冒充行业热点
- 返回“暂无高相关行业热点”，并降级到：
  1. 行业经营问题
  2. 行业消费趋势
  3. 行业常见痛点

## 3. 输出规则
每条建议至少包含：
- hotspot：热点标题
- relevance_score：行业相关分
- matched_keywords：命中的行业关键词
- reason：为什么和这个行业有关
- topic：可以怎么写成内容
- angle：建议切入角度
- action：建议动作

## 4. 禁止规则
以下情况直接判为不合格结果：
- 纯时政/外交/娱乐大盘热点，且与行业无明确关系
- 只有“热”，但没有行业词命中
- 靠模型硬解释“这个热点也许适合某行业”
- 用泛热点凑数

## 5. 一句话标准
用户选“餐饮”，结果里就应该大概率看到：
- 餐饮
- 门店
- 客流
- 外卖
- 团购
- 火锅/奶茶/咖啡

而不是：
- 王毅外长
- 国际局势
- 泛娱乐热搜

看到后者，说明检索逻辑就是错的。
