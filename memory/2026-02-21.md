## nano-banana-pro / Gemini 出图问题排查（14:00-14:30 EST）

**现象**：NianmoBot 调用 nano-banana-pro 出图时报 `500 Failed to retrieve proxy group`，反复重试无效。

**排查过程**：
1. 确认 `openclaw.json` 配置正确：`GEMINI_API_KEY=980706Aa`, `GEMINI_BASE_URL=https://load.201452.xyz/proxy/gemini`
2. 代理本身可达（文本模型 gemini-2.0-flash 正常返回 200）
3. 图片模型 gemini-3-pro-image-preview 返回 429 配额耗尽（免费层 limit=0）
4. NianmoBot 最终定位：gateway 进程是用旧配置启动的，进程环境变量残留旧的无效 `GEMINI_API_KEY=sk-TKEu3m...`，覆盖了 json 里的正确 key

**根因**：gateway 未重启，旧环境变量覆盖新配置。

**解法**：`openclaw gateway restart` 让进程加载新配置。

**经验**：改了 `openclaw.json` 里的 env 配置后，必须 `openclaw gateway restart` 才能生效。进程不会热加载环境变量。

**注意**：免费层 gemini-3-pro-image 配额可能为 0（需付费或换 key）。当前中转 `load.201452.xyz` 对 429 错误的包装方式可能误导为"proxy group"问题。
