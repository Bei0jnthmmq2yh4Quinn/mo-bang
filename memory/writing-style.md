# 老板写作风格档案

> 通过对比"小墨初稿 vs 老板改写版"持续学习，逐步逼近老板真实表达习惯。
> 每次老板口述/改写后更新此文件。

## 学习轮次记录

### 第1轮（2026-02-22）
- 主题：旅行社小红书没咨询
- 对比：小墨终稿 vs 老板口述版
- 差异分析见下方各项

## 语气基调
- **像面对面跟人聊天**，不是在"写文章"
- 不端着、不教人、不训话
- 用"讲句实话"开头，不用"说句难听的"（后者太刻意）
- 自然、随意、但有逻辑
- 像一个做了很多年的老手在跟你掏心窝子

## 用词习惯
- "比方说"（不用"比如"）
- "就是说"（口语连接词，自然出现）
- "我见过最多的就是…"（用个人经验带出观点）
- "但是"做转折（高频）
- "你要给人一种感觉"
- "没有意义"（直接否定，不绕弯）
- "恨不得"（夸张表达）
- "讲句实话"（开场白）

## 句式节奏
- 短句为主，但不是刻意每句都短
- 会有自然的长句（口语式的，带"就是说""然后"串联）
- 不用排比、不用对仗、不刻意造金句
- 金句是自然冒出来的，不是设计出来的

## 喜欢用的手法
- **俗语/古话**："兵不在多，在于精"
- **生活化比喻**："把饭嚼碎了喂人家嘴里"、"东一榔头西一榔头"
- **逆反心理解释**：解释"为什么劝退有效"时会讲心理机制
- **反问**："人家要咨询什么呢？"
- **个人经验背书**："我见过最多的…"、"我之前就经常讲…"

## 内容偏好（老板会加但小墨没写的）
- 互联网信息过载 → 用户"懒"的心理
- 客户分层（低端/高端）→ 但共同点是"怕"
- 逆反心理的解释（为什么劝退能吸引人）
- "私信服务组件"的具体提法
- 内容排期 → 重复拉优势内容做引流点（不要追热点）
- 数据复盘：看哪些内容有效，收集起来分析
- "企业号最终目的就是获客变现"
- 结尾哲学："本质不是卖服务，是帮别人解决问题"

## 小墨常写但老板会砍的
- 过于结构化的"第一步/第二步/第三步"（老板更自然流动）
- 刻意造的金句（如"收藏不等于信任"——太设计感）
- 具体的"3套模板"框架（老板用更有机的"重复拉优势内容"替代）
- 过于戏剧化的开头（"说句难听的""别发了！"）
- 严格的分段小标题

## 开头风格
- "讲句实话" + 直接切入核心问题
- 不需要炸裂标题党，但要一句话说清楚这篇在讲什么
- 用古话/俗语做支撑（如"兵不在多在于精"）

## 结尾/CTA风格
- 不喜欢硬CTA（"私信发【诊断】"这种太销售）
- 更喜欢用一句"道理"收尾
- 结尾要有"格局感"：上升到本质（"不是卖服务，是帮别人解决问题"）
- 用生活化比喻收束（"把饭嚼碎了喂人家嘴里"）

## 排版偏好
- 自然分段，不要太多分隔线
- 不用emoji装饰
- 不用markdown标题符号
- 段落之间靠空行呼吸，不靠符号

## 口语特征（第2轮 2026-02-23，语音转文字学习）
- 语音转文字 = 最接近老板真实表达的素材
- "我觉得"高频出现，表达个人判断
- "就是说"做连接词，串联上下文
- "然后"做转折/递进，一句话可以很长但不断气
- "可能还是…会比较好一点"——委婉表达偏好，不是命令式
- "对吧？""对不对？"结尾寻求共识
- "其实就很水"——直接但不刻薄的自嘲/评价
- "嘛"做语气软化："多充实一点嘛"
- 语音特点：一口气说完一个完整想法，中间不断句，靠"然后""就是说""但是"串起来
- 思维方式：先说问题 → 再说方案 → 再解释为什么 → 最后确认共识
- 和正式写作的区别：口语更长更散，但写成文案时需要保留这个"聊天感"同时收紧结构

## 口语特征（第3轮 2026-02-23，完整口述接单帖）
- "说实在的"开头（比"讲句实话"更随意）
- "直白点说"做强调转折，高频使用
- "不是…而是…"对比句式，点明真正问题
- "然后怎么怎么样"快速略过不重要的细节
- "唉"做模拟对方语气的叹词
- "事儿没干好""啥也没搞成"——口语化总结，不用书面语
- "心态爆炸""定位稀碎"——年轻化网络用语自然混入
- "说多不多，说少不少，但是刚好…"——三段式过渡
- "不是说…你不认可…你还非得…"——模拟冲突场景，有代入感
- "螺旋性上升"——偶尔蹦出专业词但用在口语语境里
- 核心节奏：抛问题 → 举反面例子 → 给正确做法 → 再加一个"还有一个点" → 聊合作门槛 → 收尾
- 段落衔接全靠"然后""所以""其实""比方说"，不用任何书面连接词
- 结尾不煽情，"合适咱再聊，不合适咱也没办法弄，对不对？"——极其自然的收束

## 模型对比结论（第4轮 2026-02-28，选题"代运营价格差异"）
- **Grok-4 胜出**：场景感最强、细节最多（举具体例子、具体数字）、节奏最像老板说话
- **GPT-5.2**：结构清晰但偏"合伙人"视角，略书面
- **DeepSeek**：完整但偏平，举例不够具体
- **Gemini**：最长最细但偏"解释型"，不像聊天
- **Claude**：风格最接近老板口语，但细节/案例不如 Grok 丰富

### Grok 写法特征（老板认可的）
- 开头直接切"报价差得离谱"，不铺垫不客套
- 分价位拆解时用"先说…再往上走…那…呢？"自然过渡
- 举例极具体："一个美妆号连续一个月""每天20-30分钟""0赞0收藏0评论"
- 自我介绍段不吹不谦："只接两三个号""宁可少赚也不浪费时间"
- 结尾留口但不推销："合适咱再聊，不合适也没关系"
- "掰扯掰扯""最扎心的是""瞎搞"——偏北方口语，老板接受

### 内容红线
- 不写自己不做的事（比如"做矩阵""刷量"）——会误导客户以为你提供这些服务
- 编造案例要基于真实案例改编，不能凭空捏造行业和数据
- 经验年限可以写（"快四年了"），老板确认过

## 账号自述画像（来自已发布笔记，2026-02-28 抓取）

### 人设定位
- 5年新媒体运营 / 4年小红书运营经验
- 目前只做小红书，不做其他平台
- 没有团队/公司，就"我和朋友两个人在做"
- 收费 3000/月，按月合作，满意再继续
- 擅长实体获客和企宣（高客单价、高净值行业）
- 不接低客单价品类（手机壳、钢化膜之类）

### 核心态度（反复出现的表达）
- **不保证效果**："你们把我当个人吧，我又不是神""我决定不了一个产品的成败"
- **不割韭菜**："对韭菜没兴趣""割一茬韭菜对我有什么意义呢"
- **不做老板IP**：成本太高（上镜+脚本+拍摄+剪辑），短期难见效，一般不推荐
- **不一步登天**："运营不是一下能做起来的，要是能一下做起来就没有穷人"
- **不试稿**："本来就不是一两篇笔记能决定的事儿"
- **长期合作导向**："本质上希望能长期合作，互利共赢"

### 服务流程（从笔记提炼）
1. 先做数据调研、行业调研 → 出运营方案
2. 确定商业定位（品牌宣传 / 引流获客 / 老板IP）→ 选一条死磕
3. 找对标账号（20个）→ 拆爆点：标题/封面/第一句话/近30天高频内容
4. 建选题分类（案例/避坑/报价拆解/科普）→ 按分类发
5. 差异化：主题细分 或 表达方式差异化
6. 主页优化：昵称/头像/简介/头图/合集 → 3秒让人知道你是谁
7. 复盘：每10篇看曝光趋势、收藏评论稳定性、私信精准度

### 数据认知
- 自然流日均稳定5-10个客资 = "能力到顶了"
- 企业号爆款10万+ 涨粉可能也就三四百，但价值在搜索长尾
- 个人号涨粉快但客资差，企业号看着慢但咨询实打实
- 一条没起来很正常，看的是能不能持续优化递增

### 高频金句/口头禅（已发布笔记中）
- "多说无益，介意勿扰"
- "时间宝贵，非诚勿扰"
- "说实在的""讲句实话""直白点说"
- "得不偿失，得不偿失！"（重复强调）
- "专业的事交给专业的人，这个是铁律"
- "不是卖服务，是帮别人解决问题"
- "合适咱再聊，不合适也没关系"

### 写作结构特征（已发布笔记）
- 用"-"或"."做分隔符，不用 markdown 标题
- 段落极短，一个想法一段
- 偶尔用编号（01/02/03），但不是每篇都用
- 话题标签放最后，5-10个
- 开头直接亮身份+经验年限，不绕弯

### 笔记结构模板（从高赞笔记提炼）

**A. 接单帖结构（373赞/79赞共性）：**
1. 身份开场 — 直接亮经验年限+只做小红书+团队规模（"5年新媒体，4年小红书，两个人做"）
2. 核心论点 — 一句话点明这篇要说什么（"聊一下为什么我是3000一个月"）
3. 价值解释 — 我做的不只是出笔记，是整体运营方案+调研
4. 管理预期 — 明确说不能做什么（"我又不是神""保证不了""能力有限"）
5. 拒绝清单 — 不试稿/不对赌/不接不懂的行业/不接离谱目标
6. 能力边界 — 给一个现实的数字（"自然流日均5-10个客资已经到顶"）
7. 结尾模块

**B. 行业观点帖结构（"代运营这行比你想的乱""被代运营坑过的人"共性）：**
1. 痛点开场 — 直接抛一个大家都有的困惑/离谱现象
2. 反面拆解 — 先说坑在哪/别人怎么忽悠你（"保证"是怎么玩文字游戏的）
3. 能做什么 vs 不能做什么 — 对比段，管理预期
4. 正面输出 — 正确做法/建议
5. 结尾模块

**C. 毒舌吐槽帖结构（"主动找你的代运营不靠谱"）：**
1. 现象引入 — 观察到的一个荒谬现象
2. 反面论证 — 用极端逻辑推导出"这不靠谱"
3. 毒舌金句 — 夹带幽默的狠话（"给你卖保健品"）
4. 正面建议 — 虽然毒舌但最后给实在建议
5. 企业号干货 — 顺带输出一段专业内容
6. 结尾模块

**D. 干货教程帖结构（"6种笔记结构""公司账号怎么运营"）：**
1. 痛点共鸣开场 — "你是不是也这样"
2. 一句话转折 — "不是你不努力，是形式选错了"
3. 编号拆解 — 01/02/03… 每个配举例
4. 行动指南 — "三步立刻用"
5. 评论区引导 — "你做什么领域？"（不用结尾模块，用互动收束）

**共性规律：**
- 高赞帖都有"先说反面再说正面"的结构——先拆别人的坑，再讲自己的做法
- "管理预期"段是必须的——不保证、不一步登天、能力有限
- 给一个具体数字比说一堆废话强——"日均5-10""涨粉三四百""做了几十个号成功60%"
- 结尾不是一句话是一个段落模块

### 结尾模式（高赞笔记共性，极重要）
结尾不是一句话收束，而是一个**固定段落模块**，包含以下元素（顺序可调）：
1. **擅长领域**："擅长实体获客和企宣这一块"
2. **服务承诺**："对每一篇笔记负责""用心去做"
3. **合作方式**："按月合作，满意再继续" / "按月收费"
4. **适用人群筛选**："高客单价，高净值人群" / 不适合的品类举例
5. **收费**："3k/月" / "3000/月"
6. **引导**："服务内容看置顶" / "合作可以看置顶笔记"
7. **收束金句**："时间宝贵，非诚勿扰" ← 几乎每篇都有

**注意**：高赞结尾从不用"合适咱再聊，不合适也没关系"——这个是口语聊天用的。发布稿结尾用"时间宝贵，非诚勿扰"更符合账号调性。

### 额外风格细节（从新拉取的笔记补充）
- **毒舌/自嘲混用**："方便等你老了，我给你卖保健品""脑子坏掉啦"——敢说敢怼，但带幽默感不伤人
- **"惟手熟尔"**——偶尔蹦文言文/成语，反差萌
- **反面案例更有力**：与其说"我能做什么"，不如先说"别人坑你的方式"
- **"能做什么 vs 不能做什么"对比段**：高赞笔记都有这个结构——先管理预期再谈合作
- **"保证"是敏感词**：老板反复强调不保证效果，凡是写"保证XX"的代运营都是骗子
- **陪跑/试稿/对赌 = 拒绝**：明确不接这三种合作方式
- 做过的行业列举：家政、旅游、摄影（实体获客为主）
- **企业号玩法公式**：关键词→搜索流量→精准人群→持续输出→小爆款→投放→周而复始
